## âš ï¸ Troubleshooting

### Virtual Environment Path Error

If you see an error like:
```
Fatal error in launcher: Unable to create process using '...\venv\Scripts\python.exe'
```

**Solution:** Delete the `venv` folder and recreate it:

```powershell
# Deactivate if active
deactivate

# Delete old venv
Remove-Item -Recurse -Force venv

# Create new venv
python -m venv venv

# Activate and install
.\venv\Scripts\Activate
pip install -r requirements.txt
```

### Port Already in Use

If a port is already in use, you can specify a different port:

**Backend:**
```powershell
uvicorn main:app --reload --port 8001
```

**Frontend:**
```powershell
npm run dev -- -p 3001
```

### npm Vulnerability Warnings

Vulnerability warnings during `npm install` are common and usually from dev dependencies. For development, you can ignore them or run:

```powershell
npm audit fix
```
